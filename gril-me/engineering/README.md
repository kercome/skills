# 🛠️ Engineering Skills Directory

> 来源: https://github.com/mattpocock/skills/tree/main/skills/engineering
> 共 10 个技能

## 目录结构

```
engineering/
├── diagnose/                              # 硬 bug 诊断与性能回归调试
├── grill-with-docs/                       # 带文档上下文的方案拷问
├── improve-codebase-architecture/         # 代码架构深化与重构
├── prototype/                             # 快速原型验证（逻辑/UI）
├── setup-matt-pocock-skills/              # 技能初始化配置
├── tdd/                                   # TDD 红-绿-重构循环
├── to-issues/                             # 计划 → 可领取 Issue
├── to-prd/                                # 对话上下文 → PRD
├── triage/                                # Issue 分诊状态机
└── zoom-out/                              # 宏观视角与上下文概览
```

## 技能概述

| # | 技能 | 描述 | 文件数 | 触发词 |
|---|------|------|--------|--------|
| 1 | diagnose | 硬 bug 诊断循环：复现→最小化→假设→仪表→修复→回归测试 | 2 | "diagnose this", "debug this" |
| 2 | grill-with-docs | 基于项目领域模型的方案拷问，同步更新文档 | 3 | "grill with docs", 压力测试计划 |
| 3 | improve-codebase-architecture | 发现架构深化机会，提出重构候选 | 5 | "improve architecture" |
| 4 | prototype | 构建可丢弃的原型验证设计（终端逻辑 / UI 变体） | 3 | "prototype this" |
| 5 | setup-matt-pocock-skills | 为仓库配置 issue tracker、标签映射、领域文档 | 6 | 首次使用工程技能前运行 |
| 6 | tdd | 红-绿-重构 TDD 循环，集成测试风格 | 6 | "TDD", "red-green-refactor" |
| 7 | to-issues | 将计划/PRD 拆分为垂直切片的实现 Issue | 1 | "create issues" |
| 8 | to-prd | 将当前对话上下文综合为 PRD | 1 | "create a PRD" |
| 9 | triage | Issue 分诊：分类→状态流转→Agent Brief | 3 | "triage issues" |
| 10 | zoom-out | 请求 AI 提供高层次上下文和模块地图 | 1 | "zoom out", 不熟悉的代码区域 |
