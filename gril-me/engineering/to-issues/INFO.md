# to-issues — 计划转 Issue

| 项目 | 内容 |
|------|------|
| **来源** | `skills/engineering/to-issues/` |
| **格式** | PromptScript |
| **触发词** | 将计划/PRD 转换为实现 Issue |
| **文件** | SKILL.md（单文件） |

## 流程

1. **收集上下文** — 从对话中获取计划内容
2. **探索代码库** — 了解当前状态（可选）
3. **起草垂直切片** — 将计划拆分为端到端的垂直切片 Issue，每片贯穿所有集成层
4. **询问用户** — 展示拆分方案，确认粒度、依赖关系、HITL/AFK 标记
5. **发布 Issue** — 按依赖顺序发布到 issue tracker，使用 Issue body 模板

## 切片规则

- 每个切片交付一个狭窄但完整的端到端路径（schema → API → UI → 测试）
- 完成的切片可以独立演示或验证
- 倾向多而薄的切片，少而厚的切片
- 切片标注 HITL（需要人工）或 AFK（可完全由 Agent 完成）
