# prototype — 快速原型验证

| 项目 | 内容 |
|------|------|
| **来源** | `skills/engineering/prototype/` |
| **格式** | PromptScript |
| **触发词** | "prototype this", "let me play with it", "try a few designs" |
| **文件** | SKILL.md, LOGIC.md, UI.md |

## 两条分支

| 分支 | 问题 | 产出 |
|------|------|------|
| **LOGIC** | "这个逻辑/状态模型对吗？" | 小型交互式终端应用，推动状态机走过纸上难以推理的场景 |
| **UI** | "这个应该长什么样？" | 多个完全不同的 UI 变体，通过 URL 参数切换，浮动底栏选择 |

## 核心规则

1. 从第一天起标记为可丢弃的，不投入生产质量
2. 一键运行（遵循项目的任务运行器惯例）
3. 默认无持久化，状态在内存中
4. 跳过润色（无测试、无错误处理、无抽象）
5. 每次操作后展示完整状态
6. 完成后删除或吸收已验证的决策
