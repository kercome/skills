# SSIM 三层分流算法详解

## 核心逻辑

工具对视频切片图片做 **SSIM (结构相似度)** 比对，分三档处理：

```
               ┌─────────────────────────┐
               │   当前帧 vs 上一个保留帧  │
               │   计算 SSIM (0.0 ~ 1.0) │
               └───────────┬─────────────┘
                           │
            ┌──────────────┼──────────────┐
            │              │              │
         SSIM ≥ 0.98   0.70~0.98     SSIM < 0.70
            │              │              │
            ▼              ▼              ▼
      ┌──────────┐  ┌──────────┐  ┌──────────┐
      │ Tier 1   │  │ Tier 3   │  │ Tier 2   │
      │ 绝对重复  │  │ 灰色地带  │  │ 绝对新页  │
      │ 直接剔除  │  │ OCR 裁决  │  │ 直接保留  │
      └──────────┘  └─────┬────┘  └──────────┘
                          │
                   文本相似度 ≥ 0.90?
                     │         │
                    是         否
                     │         │
                     ▼         ▼
                  剔除      保留
              (内容未变)  (局部更新)
```

## 三层阈值说明

| Tier | SSIM 范围 | 判 决 | 频率 | 说明 |
|------|----------|-------|------|------|
| 1 | ≥ 0.98 | 绝对重复 → 删 | 最高 | 静止画面、光标闪烁、压缩噪点 |
| 2 | < 0.70 | 绝对新页 → 留 | 中等 | 翻页、切换窗口、大幅滚动 |
| 3 | 0.70~0.98 | 灰色地带 | 最低 | 局部文字更新、轻微动画 |

## 关键设计决策

### 1. 比较基准是"上一个保留的图片"，不是前一张

`last_img_path` 只在图片被判定为"保留"时才更新。这样设计是为了防止"慢漂移"——如果跟紧挨着的前一张比（包括已被剔除的），在连续多帧之间缓慢变化时会产生误判。

### 2. OCR 延迟加载 (Lazy Loading)

基准帧的 OCR 文本不会在遍历开始时提取，而是等到第一次进入 Tier 3（灰色地带）时才触发 `extract_text()`。大部分切片走 Tier 1 或 Tier 2 就结束了，不需要 OCR，这样可以省掉大量时间。

### 3. CJK 路径兼容

OpenCV 的 `cv2.imread()` 在 Windows 上无法处理含中文/日文的路径。工具使用 `imdecode` 绕行。

### 4. 文本相似度计算

使用 Python 标准库 `difflib.SequenceMatcher`，先去除所有空白字符防止排版干扰。

## 可调参数

| 参数 | CLI 标志 | 默认值 | 调优建议 |
|------|---------|--------|---------|
| 严格 SSIM | `--strict-ssim` | 0.98 | 翻页带平滑动画可调低至 0.95 |
| 宽松 SSIM | `--loose-ssim` | 0.70 | 硬切翻页保持 0.70，柔和过渡可调至 0.60 |
| 文本阈值 | `--text-thresh` | 0.90 | 代码演示调低至 0.80，PPT小字变化调高至 0.95 |
