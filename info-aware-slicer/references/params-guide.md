# info_aware_slicer.py 参数调优指南

## 全部 CLI 参数

| 参数 | 默认值 | 范围 | 作用 |
|------|--------|------|------|
| `--video` | 必填 | — | 输入视频路径 |
| `--output-dir` | 必填 | — | 输出目录（切片图 + document_chain.json） |
| `--sad-threshold` | 8.0 | ≥ 0 | SAD 运动触发阈值，越低越敏感 |
| `--template-match-thresh` | 0.75 | (0, 1] | 尾首模板匹配相关系数阈值 |
| `--overlap-ratio` | 0.15 | (0, 0.5] | 取上一帧底部作为模板的高度比例 |
| `--base-step-sec` | 0.3 | ≥ 0 | NEW_PAGE 后默认前进量（秒） |
| `--max-step-sec` | 1.0 | ≥ 0 | 低运动/静止时最大跳跃（秒） |
| `--still-sad` | 2.0 | ≥ 0 | 静止判定 SAD，低于此值跳过当前帧 |

## 参数调优指南

### 1. `--sad-threshold` — 运动触发敏感度

决定"画面有没有变化"。

| 场景 | 建议值 |
|------|--------|
| 画面清晰、内容变化明显 | 6.0 ~ 8.0 |
| 有轻微噪点/压缩伪影 | 10.0 ~ 15.0 |
| 慢速滚动，想切得密一些 | 4.0 ~ 6.0 |
| 只要大幅翻页，忽略小滚动 | 12.0 ~ 20.0 |

**判断标准**：切片太多（一页被切成多张）→ 阈值太低；切片太少（遗漏变化）→ 阈值太高。

### 2. `--template-match-thresh` — 连续滚动判断

决定"上一帧底部和当前帧顶部是不是同一内容"。

| 场景 | 建议值 |
|------|--------|
| 标准文档滚动 | 0.70 ~ 0.75 |
| 文本密集、变化细微 | 0.65 ~ 0.70 |
| 快速翻页，严格区分翻页 vs 滚动 | 0.80 ~ 0.85 |

### 3. `--overlap-ratio` — 重叠检测区域

取前一帧底部多少比例匹配当前帧顶部。

| 场景 | 建议值 |
|------|--------|
| 中等滚动速度 | 0.15 |
| 飞快滚动、帧间重叠小 | 0.08 ~ 0.12 |
| 慢速滚动、大量重叠 | 0.20 ~ 0.30 |

### 4. `--base-step-sec` — 基础步长 ⭐

**这是最重要的参数。** NEW_PAGE 后每跳前进的秒数。

| 场景 | 建议值 |
|------|--------|
| 慢速滚动文档 | 0.15 ~ 0.20 |
| 中速翻页（默认） | 0.25 ~ 0.35 |
| 快速翻页、内容跳跃大 | 0.40 ~ 0.60 |

**核心原则**：如果人工发现两张切片之间缺内容超过半秒 → 减小此值。

### 5. `--max-step-sec` — 最大步长 ⭐

低运动时的大跳。默认 1.0s 替代了原来的 2.0s。

| 场景 | 建议值 |
|------|--------|
| 慢速滚动、不希望漏帧 | 0.5 ~ 0.8 |
| 中速翻页 | 1.0 |
| 静止段很多、想加速 | 1.5 ~ 2.0 |

### 6. `--still-sad` — 静止判定

低于此 SAD 的帧不在切片中保留。

| 场景 | 建议值 |
|------|--------|
| 60fps 高帧率（帧间差异小） | 1.0 ~ 1.5 |
| 30fps 标准帧率 | 2.0 |
| 画面噪点多、容易误判静止 | 0.5 ~ 1.0 |

## 预设场景组合

### 精确模式（几乎不漏帧）

```
--sad-threshold 4.0 --template-match-thresh 0.70 --overlap-ratio 0.12
--base-step-sec 0.15 --max-step-sec 0.5 --still-sad 1.0
```

### 默认模式（平衡）

```
--sad-threshold 8.0 --template-match-thresh 0.75 --overlap-ratio 0.15
--base-step-sec 0.3 --max-step-sec 1.0 --still-sad 2.0
```

### 快速模式（只保留关键翻页）

```
--sad-threshold 15.0 --template-match-thresh 0.82 --overlap-ratio 0.10
--base-step-sec 0.5 --max-step-sec 2.0 --still-sad 3.0
```

## 输出解读

切片完成后，查看 `document_chain.json` 的 status 分布：

| status | 含义 | 正常比例 |
|--------|------|---------|
| `FIRST_FRAME` | 视频首帧 | 1 个 |
| `CONTINUOUS` | 连续滚动（尾首匹配成功） | 滚动场景多 |
| `NEW_PAGE` | 硬切换/翻页 | 翻转场景多 |
| `MISSING_GAP` | 可能丢帧（匹配失败 + 运动大） | 少量 |

如果 `CONTINUOUS` > 50% 但实际上是翻页 → 提高 `--template-match-thresh`。
如果 `MISSING_GAP` > 20% → 降低 `--template-match-thresh` 或减小步长。
如果 `NEW_PAGE` 几乎 100% → 这是正常的（文档视频以翻页为主）。
